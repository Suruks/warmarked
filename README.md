# Warmarked — прототип v1 (Godot 4.7)

Играбельный hotseat/pass-and-play прототип боевой петли из дизайн-дока
(`blind_wego_tactics_design_1.md`). Графики нет — примитивы и текст.

## Запуск

Открыть проект в **Godot 4.7** и нажать Play (F5), либо из командной строки:

```
"C:\SteamLibrary\steamapps\common\Godot Engine\godot.windows.opt.tools.64.exe" --path E:\dev\warmarked
```

## Как играть (два игрока за одним устройством, портретный экран)

1. Экран **«Передайте устройство игроку X»** → нажать «планировать».
2. **Тап по фигуре на поле** — под доской показываются скиллы этого юнита (можно тапнуть
   и вражеского — только просмотр). **Тап по своему скиллу** → использование: если нужна
   цель, кликните подсвеченную клетку. Действие занимает активный слот (внизу ряд «Слоты»),
   активный слот авто-переходит к следующему. «Пропустить слот» — оставить пустым (банк маны/тайминг).
   Нацеленные скиллы помечаются мелкими иконками прямо в целевых клетках (напоминание).
3. «Готово». Устройство передаётся второму игроку (его приказы не видны первому).
4. **Разрешение** проигрывается с анимацией (ход — скольжение, ближняя атака — толчок,
   дальняя — белая вспышка атакующего → красная у цели), лог справа. В конце — «Завершить раунд».
5. Первым набравший **7 очков** выигрывает (килл +1, большинство точек +1/раунд).
   Килл — это темп, а не победа: пока враг лежит, вы и так забираете точки.

На доске: круглые иконки — герои (кольцо синее = игрок A, красное = B), под ними `HP·мана`.
Золотые ромбы — точки контроля. Тёмные клетки — террейн. `X` — капкан, `!` — засада.
Голубое кольцо — щит, оранжевое — обездвижен. Полупрозрачная иконка — призрак запланированного хода.

## Сетевая игра (авторитетный headless-сервер)

Транспорт — WebSocket, встроенный в Godot (Photon не нужен). Сервер авторитетно ведёт
матчмейкинг и **держит приказы обоих игроков скрытыми, пока не пришли оба** (слепой гейт),
затем раскрывает. Игра детерминирована, поэтому по сети гоняются только 4 приказа/раунд —
клиенты приходят к идентичному результату (лок-степ).

**Запуск сервера** (один бинарь на VPS/локально):
```
"…\godot.windows.opt.tools.64.exe" --headless --path E:\dev\warmarked -- server
```
Слушает `ws://0.0.0.0:8910`.

**Клиент**: запустить игру, в меню ввести адрес сервера (по умолчанию `127.0.0.1`) и нажать
«Онлайн (найти игру)». Когда в очереди двое — создаётся матч, дальше каждый планирует только
своего игрока, жмёт «Готово» → приказы уходят на сервер → после хода соперника раунд
проигрывается с анимацией у обоих.

Клиенту не доверяем: приказы проходят через [net_protocol.gd](scripts/net/net_protocol.gd)
(типовая валидация — битый пакет не роняет сервер) и
[order_validator.gd](scripts/model/order_validator.gd) (легальность: дальность хода, геометрия
цели, мана, гейт слотов, «один скилл не дважды»). Санитизация происходит **до** сохранения
приказа, поэтому раскрывается ровно то, что резолвится, и лок-степ не расходится.

Слои: [net.gd](scripts/net/net.gd) (автолоад `Net`: сокеты/матчмейкинг/RPC),
[match_session.gd](scripts/net/match_session.gd) (авторитетная сессия матча, слепой гейт),
[net_protocol.gd](scripts/net/net_protocol.gd) (сериализация приказов).

## Архитектура

- `scripts/model/` — **чистая логика** (без узлов сцены), тестируется headless:
  `consts` (все дайлы баланса), `board`, `unit`, `hero_defs`, `order`, `targeting`,
  `resolver` (разрешение раунда), `match_state` (петля/мана/очки/респ),
  `order_validator` (авторитетная проверка легальности приказов).
- `scripts/ui/` — `board_view`, `planning_panel`, `resolution_view`, `main` (автомат фаз).
- `main.tscn` — корневая сцена.

## Дайлы баланса

Все крутибельные числа — в `scripts/model/consts.gd` (HP, мана/урон способностей,
`WIN_SCORE`, `KILL_POINTS`, `RESPAWN_DELAY`, `COLLISION_DMG`,
`PERSIST_ROUNDS` и т.д.). Раскладка карты (террейн, точки, старт) — в
`board.gd::_build_default_map` и `match_state.gd::setup`.

## Интерпретации спорных мест (в доке помечены [ASSUMPTION]/[RISK])

Реализованы конкретно — можно менять:

- **No-retarget:** при планировании фиксируется не абсолютная клетка, а **смещение** цели от
  героя (направление и дистанция). В момент разрешения эффект целит `текущая_клетка + смещение`,
  поэтому отброс/сдвиг героя до его действия уносит и точку прицела — план остаётся тем же
  жестом, а не «умной» самонаводящейся атакой. Бьёт того, кто окажется в этой клетке
  (**друг ИЛИ враг**), иначе физзлит. Исключение — Лечение и Отмена (только союзник) и
  Прыжок (урон только врагу). Поле `Order.relative` — этот режим; `Order.target` хранится
  только для отрисовки маркеров.
- **Движение:** союзники не блокируют — сквозь них проходят насквозь, но встать на их клетку
  нельзя (две фигуры не стоят на одной). Враг блокирует полностью, даже как транзит. Транзитная
  клетка не считается «входом», поэтому капканы/засады на ней не срабатывают.
- **LOS выстрелов** проверяется только по террейну (статичен), не по юнитам.
- **Вспышка** = урон в радиусе 1 по Chebyshev (8 клеток), Фея в центре не задета.
- **Отмена** (щит) — цель себе ИЛИ союзнику рядом (Chebyshev 1); гасит следующий эффект по цели.
- **Дробь**: целит диагональную клетку T, бьёт T + две орто-соседние с дальней стороны,
  отбрасывание 1 от стрелка (в препятствие/край → +урон столкновения).
- **Капкан/Засада** живут раунд размещения + `PERSIST_ROUNDS` (сейчас 0 → ровно 1 раунд), одноразовы.
  Засада, как и капкан, срабатывает только на вход **вражеского** юнита в клетку рядом
  (Chebyshev 1) с Кристалкайндом и следует за ним; союзники её не задевают.
- **Обездвиживание** от капкана гасит ходы жертвы в СЛЕДУЮЩЕМ раунде.
- **Мана:** старт 1, доход +1 со 2-го раунда (в сумме «старт 1, +1/раунд»). Мёртвый ману
  не копит; респ — полное HP и мана, сброшенная к `START_MANA` (накопленный банк не переживает смерть).
- **Смерть и респ:** убитый в раунде N возвращается в начале раунда `N + RESPAWN_DELAY`, т.е.
  пропускает `RESPAWN_DELAY - 1` полных раундов (сейчас — один). Воскресает в **родной клетке**
  (старт), а если она занята — в ближайшей свободной своего ряда. На клетке смерти остаётся
  могила, но она никого не блокирует и никого не наказывает. Реальная цена смерти — потерянный
  темп плюс сброшенный банк маны.
- **Первый игрок**: по чётности раунда (нечёт → A). Жеребьёвка — поле `a_first_on_odd`.
